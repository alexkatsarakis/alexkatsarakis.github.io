Object Manager

Description:

Object Manager is responsible for keeping track of everything related to the objects. First, it is able to save individual RenderManagers (check RenderManagers tab for more info) that each one will provide their constructor for objects. Secondly, it will keep track of the constructors available at any point. And third, it keeps track of all the objects alive.


Info:

  • Path: {src}/Engine/renderer/ObjectManager.js
  • Exports: ObjectManager objectManager
  • Responsible to initialize: ObjectManager
  • API access: Engine.ObjectManager

API Exported:

    Most frequent:
  1. objects: Return an object that contains all the alive objects currently on the game.
  2. getObjectByID(objId): Return the object corresponding to this id if it's alive.
  3. getObjectByName(objName): Return the object corresponding to this name if it's alive

    4. Less frequent:
  4. isSystemObject(objId): Return true/false if the object is listed as systemObject (like scene,collision,keyboard).
  5. addSystemObject(objId): From now on isSystemObject(objId) will return true for this id.
  6. renderAll(): Will render all the objects to the screen. (it's called by Engine each loop)
  7. addToWorld(newObject): Add newObject to the world. (usually called within the initialisation of the object itself)
  8. removeFromWorld(aliveObject): Removes aliveObject from the world. (usually called within the deconstruction of the object itself)
  9. addConstructor(name,constr): Installs a new constructor for objects. (installed by ObjectManager itself)
  10. removeConstructor(name): Removes an installed constructor from the manager
  11. getConstructor(name): Returns the constructor with that name.

System Objects

The Object Manager is responsible to instantiate these objects since these objects are not bound to any other manager. More information can be found on System Object tab


Example

At the moment, there are two different RenderingManagers working at the same time. One is using the browser's element system and the second one is using a custom made rendering system using canvas. So, when I install the First Manager all the constructors associated with it will come along and be installed to the system. From this point forward, the ObjectManager will be calling the render function of the manager each frame. and each object created from a constructor that belong to that manager will have to be rendered.
