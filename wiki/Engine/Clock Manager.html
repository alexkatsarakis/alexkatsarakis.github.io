Clock Manager

Description:

Clock Manager is responsible for keeping the game time and also provide some useful functionality for the user based on time.


Info:

  • Path: {src}/Engine/clock/ClockManager.js
  • Exports: Class ClockManager()
  • Responsible to initialize: Engine
  • API access: Engine.ClockManager

API Exported:

  1. update(): This function is updating the game time and keeps track of FPS internally. Must be called each frame once.
  2. callIn(callback, args, delay): Similar to setTimeout(cb,delay) but according to game timer and it triggers the callback at the end of each update if the new timer is after the delay
  3. cancelCallBack(returnedObjectOfCallIn): Similar to clearTimeout(returnedObjectOfsetTimeout).

Future Plans:

  1. getFPS(): will return the current FPS of the game. (currently served throught blackboard)
  2. getGameTime(): will return the current game time. (currently served throught blackboard)
  3. callInRepeat(callback, args, delay): will work like setInterval(cb, delay) but according to game timer.
  4. fastTime(percentage): The time will be moving percentage% faster. (By changing dt on animations).
  5. slowDown(percentage): The time will be moving percentage% slower. (By changing dt on animations).
