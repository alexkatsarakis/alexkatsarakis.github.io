Hud

Description:

Hud is the main UI for the system that is necessary if someone wants to create interact with the system.


Main Parts:

The hud consists from some small components

  1. infoBar: Shows an info above the coding area.
  2. eventsTab: Shows all the available scripts that the user can code.
  3. languageDiv: Is the div element that the corresponding language (check Scripting Manager for details) is currently running.
  4. openTab: Display the name of the script that is currently showing to the end user.
  5. playScriptButton: Clicking this button will run the current script.
  6. saveScriptButton: Will save the current script to the corresponding position.
  7. showScriptButton: When pressed will show the Javascript code that was produced by the current language.
  8. fpsCounter: Shows the current fps of the system.
  9. hudToggle: Every element that has class="hudChild" will be automatically hidden from the user and when clicked again the UIs will appear.


How does it work?

When the UI is first loaded it hooks some function to some variables on the blackboard of the system. That means, that every time this variable changes value the callback is being triggered. Having this mechanism enables the system to be more interactive for example when a user clicks on an object The value on blackboard-> state -> focusedObject is changed and the callback to re-render all the appropriate elements fires. The same happens with the fps (blackboard -> state -> FPS).
