UI Manager Architecture

Description:

UI Manager is a crucial part of the system. It is the outer most layer of the Macro-Architecture (as shown on Macro-Architecture tab) and it is the UI that the end user will interact the most. It is important to mention that, all the other sub-systems (Engine, Engine Extension, Actions) are completely independent and unaware of the UI sub-system. This gives us the ability to create UIs that are both functional and aesthetically pleasing for the end user.


The AddOn mechanisms:

Since, we established that the UIs are only interacting with the system through the provided API that means, that we can add as many UIs we want without "polluting" our base code.


API provided:

  1. installUI(args): Installs a UI to the Manager and will be a part of the system from now on.
  2. removeUI(name): Removes a UI from the Manager.
  3. loadUI(name): Load an installed UI and from now on it will be visible.
  4. hideUI(name): Hides a UI and now it won't be visible from the user.
  5. getUIs(): Returns an array of all the UIs installed at the system.
  6. getLoadedUIs(): Returns an array of all the loaded UIs currently on the system.

Features:

  1. Dynamically install,load,hide,remove UIs during the game play.
  2. Create dynamic UIs using it's own HTML5,CSS,JS with the provided API to interact with the system.
  3. Create Game Objects with HTML5,CSS,JS that will get intergrated to the game when the UI is installed for the first time. This enable the AddOn developers to do very high level UI functionality like adding videos, other web pages and other features to the game.

How do I add an AddOn?

  1. Within the UI folder of the system you create a folder with the name of the UI you want.
  2. You create a Javascript file with the same name within that folder.
  3. The Javascript must export the followings.
    export default {
      name:'{the name of the UI}',
      link: './src/UI/{folder}/{htmlFile}.html',
      cb: onLoadCallback,
      removable: true, // if the can be removed after installation
      loadOnInstall: true // if we want to load the UI right away
    };
  4. To use the API import Engine and follow each API you want (check the appropriate Manager tab)
  5. To use the blackboard import blackboard and check what things are installed at the moment.
